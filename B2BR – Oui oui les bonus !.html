  • FTP et Csftpd : Un aperçu

    FTP (File Transfer Protocol)

    C'est quoi "FTP" ? Un protocole réseau standard pour transférer des fichiers d'un ordinateur à un autre sur un réseau. Il est relativement simple à utiliser et très répandu.
    • Comment ça fonctionne ? Le client FTP se connecte au serveur FTP, puis les deux échangent des commandes pour transférer des fichiers.
    • Les limites de FTP:
      • Sécurité: Les informations, y compris les mots de passe, sont transmises en clair, ce qui les rend vulnérables aux interceptions.
      • Performance: Le protocole FTP n’est pas optimisé pour les transferts de grands fichiers ou de nombreux petits fichiers.

    Vsftpd (Very Secure FTP Daemon)

    C'est quoi "Vsftpd" ? Un serveur FTP open-source particulièrement conçu pour la sécurité. Il implémente le protocole FTP et offre de nombreuses fonctionnalités supplémentaires.
    • Pourquoi vsftpd ?
      • Sécurité: Il corrige de nombreuses vulnérabilités présentes dans d’autres serveurs FTP et offre des mécanismes d’authentification plus robustes.
      • Performance: Il est optimisé pour offrir des performances élevées, notamment pour les transferts de nombreux fichiers.
      • Flexibilité: vsftpd est hautement configurable, permettant de personnaliser les autorisations, les limites de bande passante, etc.
    • Comment ça fonctionne ? vsftpd écoute les connexions FTP entrantes et gère les requêtes des clients en fonction de sa configuration.
    En résumé

FTP est le protocole de base pour le transfert de fichiers, mais il présente des faiblesses en termes de sécurité.

Vsftpd est un serveur FTP sécurisé et performant, idéal pour mettre en place un serveur de fichiers sur un système Linux.

  • Divers / Notes

    UFW (Uncomplicated FireWall) :

    Afficher la liste des règles existante :

    sudo ufw status

    Ajouter une règle / un port :

    sudo ufw allow 4242

    Supprimer une règles / un port :

    sudo ufw delete allow 22

    Partition :

    Voir les partitions :

    lsblk

    Hostname :

    Voir les info hostname :

    hostnamectl

    Changer hostname :

    nano /etc/hostname

    Pour voir le changement :

    reboot

    La signature :

    Pour creé une signature (linux) :

    sha1sum nom_de_la_machine.vdi
    Attention a bien l'effectuer dans le fichier de votre VM avec le .vdi ou .vhd !

    Sudo :

    Pour connaitre l’adresse IP de la VM :

    hostname -I

    Pour se connecter en ssh via le port 4242 :

    ssh nom_utilisateur@IP_de_la_VM -p 4242

    Montrer que sudo est installé :

    sudo apt list sudo

    Log de sudo :

    sudo cat /var/log/sudo/sudo.log

    User et Group :

    Creé un utilisateur :

    sudo adduser new_user

    Supprimer un utilisateur :

    sudo deluser name_user

    Creé un groupe :

    sudo groupadd new_group

    Supprimer un groupe :

    sudo groupdel name_group

    Pour connaitre les utilisateurs dans un groupe :

    sudo getent group name_group

    Pour connaitre dans quel groupe l’utilisateur est :

    groups name_user

    Ou :

    sudo getent group | grep user_name

    Pour ajouter un utilisateur dans un groupe :

    sudo usermod -aG group_name user_name

    Ou :

    sudo gpasswd -a user_name group_name
    -a = ajouter et -G utilisée pour spécifier la liste des groupes auxquels l'utilisateur doit appartenir.

    Pour supprimer un utilisateur dans un groupe :

    sudo deluser user_name group_name

    Ou :

    sudo gpasswd -d user_name group_name

    SSH :

    Voir si SSH est actif :

    sudo systemctl status ssh

    Monitoring.sh

    Emplacement du script :

    cd /usr/local/bin

    Pour lancer le script :

    /usr/local/bin/monitoring.sh

    Crontab

    Modifier le crontab via les droit de root :

    sudo crontab -u root -e
    -u root spécifie que les tâches cron à éditer sont celles de l'utilisateur root et -e permet de modifier.

    Password

    Pour accéder aux règles de mot de passe et/ou les modifier :

    nano /etc/pam.d

sudo nano /etc/login.defs

    Pour accéder aux règles de mot de passe de l’utilisateur choisit :

    sudo chage -l name_user

  • C’est quoi SystemCTL
    Systemctl est un outil puissant utilisé sous Linux pour gérer les services système. Il a remplacé les anciens outils comme init et service dans de nombreuses distributions Linux modernes, notamment celles basées sur systemd.

    Qu’est-ce qu’un service système ?

    Un service système est un processus qui s’exécute en arrière-plan et fournit une fonctionnalité spécifique au système. Par exemple, le service network est responsable de la gestion du réseau, tandis que le service httpd gère un serveur web Apache.

    Les principales fonctionnalités de Systemctl :

    • Démarrer, arrêter et redémarrer des services:
      Les commandes les plus courantes sont systemctl start, systemctl stop et systemctl restart.
    • Vérifier l’état des services:
      La commande systemctl status permet d’obtenir des informations détaillées sur un service, comme son état (actif, inactif, en cours de chargement, etc.), son PID (process ID), et son journal.
    • Activer et désactiver les services au démarrage:
      Les commandes systemctl enable et systemctl disable permettent de configurer les services qui doivent démarrer automatiquement au démarrage du système.
    • Lister les services:
      La commande systemctl list-units permet d’obtenir une liste de tous les services disponibles.

    Syntaxe de base :

    La syntaxe de base de systemctl est assez simple :

    bash
    systemctl <action> <unit>
    <action> : L'action à effectuer (start, stop, status, enable, disable, etc.)
    <unit> : Le nom de l'unité de service (par exemple, network.service, httpd.service).

    Pourquoi utiliser Systemctl ?

    • Unification: Systemctl offre une interface unifiée pour gérer tous les types de services système.
    • Fiabilité: Il est conçu pour être plus robuste et plus fiable que les anciens outils.
    • Journalisation: Systemctl intègre un système de journalisation puissant qui facilite le dépannage.
    • Dépendances: Il peut gérer les dépendances entre les services, ce qui garantit un démarrage correct du système.

    En résumé, Systemctl est un outil essentiel pour tout administrateur système Linux. Il permet de gérer efficacement les services système, d'améliorer la stabilité du système et de faciliter le dépannage.

  • Zsh vs Bash : Quel shell choisir ?
    Zsh et Bash sont tous deux des shells, c'est-à-dire des interfaces en ligne de commande qui permettent aux utilisateurs d'interagir avec leur système d'exploitation. Ils servent à exécuter des commandes, manipuler des fichiers et automatiser des tâches. Bien qu'ils partagent des fonctionnalités communes, ils présentent des différences notables qui peuvent influencer votre choix.

    Bash : Le shell par défaut

    • Très répandu: Bash est le shell par défaut sur la plupart des distributions Linux et macOS.
    • Simple et efficace: Il offre une grande stabilité et une interface intuitive pour les tâches courantes.
    • Largement documenté: Grâce à sa popularité, Bash bénéficie d’une abondante documentation et de nombreux tutoriels.

    Zsh : Le shell moderne et personnalisable

    • Plus récent: Zsh a été développé plus récemment que Bash et intègre des fonctionnalités modernes.
    • Haute personnalisation: Zsh est extrêmement personnalisable grâce à des outils comme Oh My Zsh qui proposent une multitude de thèmes et de plugins.
    • Fonctionnalités avancées: Il offre des fonctionnalités plus avancées comme la complétion par tabulation intelligente, la correction orthographique et l’historique interactif.
    • Expérience utilisateur améliorée: Zsh vise à rendre l’expérience en ligne de commande plus agréable et productive.

    Tableau comparatif

    En résumé, Zsh est un excellent choix pour les utilisateurs expérimentés qui souhaitent personnaliser leur environnement de travail et bénéficier de fonctionnalités avancées. Bash reste un choix solide pour les utilisateurs qui préfèrent un shell plus simple et plus stable.

  • C’est quoi, SUDO
    Sudo est un utilitaire présent sur les systèmes Unix-like (comme Linux) qui permet à un utilisateur d'exécuter des commandes avec les privilèges d'un autre utilisateur, généralement le superutilisateur (root). C'est un moyen sécurisé de déléguer des tâches administratives sans exposer le mot de passe root.

    En d’autres termes, Sudo vous permet de « devenir » temporairement un autre utilisateur pour effectuer des actions qui nécessitent des droits d’administration.

    Pourquoi utiliser Sudo ?

    • Sécurité: Éviter de se connecter en tant que root en permanence réduit les risques de compromettre le système.
    • Flexibilité: Accorder des privilèges spécifiques à différents utilisateurs ou groupes.
    • Traçabilité: Sudo garde une trace des commandes exécutées avec des privilèges élevés.

    Comment ça marche ?

    1. Configuration: Un fichier de configuration (généralement /etc/sudoers) définit quels utilisateurs peuvent exécuter quelles commandes avec Sudo.
    2. Utilisation: Pour exécuter une commande en tant que root, vous tapez sudo suivi de la commande. Vous serez alors invité à entrer votre mot de passe.
    3. Autorisation: Si vous êtes autorisé à utiliser Sudo pour cette commande, elle sera exécutée avec les privilèges spécifiés.

    Exemple:

    Pour installer un nouveau paquet avec les droits root, vous pouvez taper :

    bash
    sudo apt install <nom_du_paquet>
    En résumé, Sudo est un outil essentiel pour gérer les privilèges sur les systèmes Unix-like. Il permet d'améliorer la sécurité et la flexibilité de l'administration système.

    Configuration pour born to be root :

    Dans le cadre de notre projet, on doit modifier la config, voici les ligne de configuration pour notre sudo :

    sudo visudo
    Defaults	env_reset
Defaults	mail_badpass
Defaults	secure_path="/usr/local/sbin:/usr/local/bin:/usr/bin:/sbin:/bin"
Defaults	badpass_message="Password is wrong, please try again!"
Defaults	passwd_tries=3
Defaults	logfile="/var/log/sudo/sudo.log"
Defaults	log_input, log_output
Defaults	requiretty
    Ligne 1 : Defaults env_reset 

Cette directive réinitialise l'environnement de chaque commande exécutée avec sudo.
Pourquoi ? C'est une mesure de sécurité qui empêche des scripts malveillants d'hériter d'un environnement potentiellement dangereux.
    Ligne 2 : Defaults mail_badpass

Cette directive envoie un email à l'utilisateur root à chaque tentative de mot de passe échouée avec sudo.
Pourquoi ? Cela permet de surveiller les activités suspectes et de détecter d'éventuelles tentatives d'intrusion.
    Ligne 3 : Defaults secure_path="/usr/local/sbin:/usr/local/bin:/usr/bin:/sbin:/bin" 

Cette directive restreint les chemins où sudo cherchera les commandes à exécuter.
Pourquoi ? C'est une mesure de sécurité qui empêche l'exécution de commandes arbitraires.
    Ligne 4 : Defaults badpass_message="Password is wrong, please try again!" 

Cette directive définit le message d'erreur affiché en cas de mot de passe incorrect.
Pourquoi ? Cela fournit une information claire à l'utilisateur.
    Ligne 5 : Defaults passwd_tries=3 

Cette directive limite le nombre de tentatives de mot de passe à trois.
Pourquoi ? C'est une mesure de sécurité pour prévenir les attaques par force brute.
    Ligne 6 : Defaults logfile="/var/log/sudo/sudo.log" 

Cette directive spécifie le fichier dans lequel les activités de sudo sont enregistrées.
Pourquoi ? Cela permet de garder une trace des commandes exécutées avec des privilèges élevés pour des raisons de sécurité et d'audit.
    Ligne 7 : Defaults log_input, log_output 

Cette directive enregistre à la fois les entrées et les sorties des commandes sudo dans le fichier de log.
Pourquoi ? Cela fournit un enregistrement détaillé des actions effectuées.
    Ligne 8 : Defaults requiretty 

Cette directive exige qu'un terminal (tty) soit attaché au processus avant que sudo n'accorde les privilèges.
Pourquoi ? Cela empêche les attaques à distance et l'escalade de privilèges accidentelle.

  • Guide pour une politique de mot de passe forte !

    Politique de syntaxe.

    D’abord on Installe la bibliothèque « Password Quality Checking Library »

    Cette commande sert à installer la bibliothèque libpam-pwquality :

    sudo apt-get install libpam-pwquality

    Ensuite on accède au regle de mot de passe avec :

    sudo vim /etc/pam.d/common-password

    Par la suite, sur cette ligne :

    /etc/pam.d/common-password
    password    requisite    pam_pwquality.so

    cette ligne sert à implementer les restrictions pour mot de passe et nous mettons cela :

    /etc/pam.d/common-password
    password    requisite    pam_pwquality.so retry=3 minlen=10 ucredit=-1 lcredit=-1 dcredit=-1 maxrepeat=3 reject_username difok=7 enforce_for_root

    Explication :

    password : mot de passe (indique qu'il s'agit d'une politique liée aux mots de passe).

requisite : requis (le respect de cette règle est obligatoire).

pam_pwquality.so : module utilisé pour vérifier la qualité des mots de passe.

retry=3 : l'utilisateur a 3 essais pour entrer un mot de passe valide.

minlen=10 : le mot de passe doit comporter au moins 10 caractères.

ucredit=-1 : le mot de passe doit inclure au moins une lettre majuscule.

lcredit=-1 : le mot de passe doit inclure au moins une lettre minuscule.

dcredit=-1 : le mot de passe doit inclure au moins un chiffre.

maxrepeat=3 : interdit les répétitions de plus de 3 caractères identiques consécutifs.

reject_username : interdit l'utilisation du nom d'utilisateur comme mot de passe.

difok=7 : le nouveau mot de passe doit être différent de l'ancien d'au moins 7 caractères.

enforce_for_root : applique ces règles au compte root également.

    Politique d’expirations.

    Tapez dans votre machine virtuelle :

    sudo vim /etc/login.defs

    Trouvez cette section :

    /etc/login.defs
    PASS_MAX_DAYS 9999 PASS_MIN_DAYS 0 PASS_WARN_AGE 7

    Modifiez-la comme suit :

    /etc/login.defs
    PASS_MAX_DAYS 30 PASS_MIN_DAYS 2 PASS_WARN_AGE 7

    Enfin, redémarrez la machine pour appliquer les changements avec :

    sudo reboot

    avec :

    PASS_MAX_DAYS 30 :

    • Définit la durée maximale (en jours) pendant laquelle un mot de passe peut être utilisé avant d’être changé.
    • Ici : un mot de passe devra être renouvelé tous les 30 jours.

    PASS_MIN_DAYS 2 :

    • Définit la durée minimale (en jours) avant qu’un utilisateur puisse changer son mot de passe après l’avoir défini.
    • Ici : un mot de passe ne peut être changé qu’après au moins 2 jours, empêchant les utilisateurs de contourner les règles en réutilisant immédiatement un ancien mot de passe.

    PASS_WARN_AGE 7 :

    • Définit combien de jours avant l’expiration du mot de passe l’utilisateur recevra un avertissement.
    • Ici : l’utilisateur sera informé 7 jours avant que son mot de passe n’expire.

    Vérification :

    Affin de vérifier les nouvelles règles de mot de passe on utilise deux outil :

    • Chage
      • La commande chage permet de gérer aisément les contraintes liées à la validité des comptes et des mots de passe. Au moyen de l’option –l, cette commande, lancée en simple utilisateur, permet d’afficher les informations de validités associées de notre compte
    • Passwd
      • Cette commande permet de modifier le mot de passe des comptes d’utilisateurs.
    Exemple pour chage.
    ~ chage -l hdruel
Last password change                    : Nov 14, 2024
Password expires                    : Dec 14, 2024
Password inactive                    : never
Account expires                        : never
Minimum number of days between password change        : 2
Maximum number of days between password change        : 30
Number of days of warning before password expires    : 7

    Attention ! Les utilisateur crée avant la modification de « /etc/login.defs » on encore l’ancienne politique d’expiration, il est donc nécessaire de la modifier manuellement via Chage aussi !

    Exemple pour passwd
    ~ sudo passwd hdruel
[sudo] password for hdruel: MotDePass42@
New password: HdruelHdruel42!
BAD PASSWORD: The password contains the user name in some form
New password: MotDePassDeFouMalade42@
Retype new password: MotDePassDeFouMalade42@
passwd: password updated successfully

    On notera que l’on peut test toutes les politique de syntaxe de mot de passe avec Passwd.

    Résumé :

    La politique de mot de passe est super importante et nécessaire dans notre projet. 

On utilise libpam-pwquality pour la syntaxe obligatoire et on modifie le fichier /etc/login.defs pour les règles d'expirations

  • C’est quoi une VM !
    Une machine virtuelle (VM) est une émulation d'un ordinateur complet au sein d'un autre ordinateur. Elle fonctionne comme un ordinateur indépendant, avec son propre système d'exploitation, ses applications et ses données. Cependant, elle n'existe que de manière logicielle et partage les ressources physiques (CPU, mémoire, disque) de la machine hôte. Les machines virtuelles sont gérées par un logiciel appelé hyperviseur. Elles offrent une grande flexibilité pour tester différents systèmes d'exploitation, logiciels et configurations réseau, sans affecter le système hôte.

    Pourquoi utiliser une machines virtuelle ?

    1. Environnement isolé et reproductible:
      • Expérimentation sans risque: Les étudiants peuvent tester de nouvelles configurations, installer des logiciels ou effectuer des modifications sans craindre d’endommager leur système principal.
      • Réinitialisation facile: En cas de problème, il suffit de restaurer une sauvegarde ou de créer une nouvelle machine virtuelle.
      • Environnements multiples: Les étudiants peuvent créer plusieurs machines virtuelles pour tester différents systèmes d’exploitation ou configurations réseau.
    2. Flexibilité:
      • Personnalisation: Les étudiants peuvent configurer leur machine virtuelle selon leurs besoins spécifiques, en choisissant le système d’exploitation, les logiciels, et les paramètres réseau.
      • Adaptation aux projets: Les machines virtuelles peuvent être adaptées à chaque projet, en simulant différents environnements de production.
    3. Portabilité:
      • Indépendance: Les machines virtuelles peuvent être déplacées et exécutées sur différentes machines physiques, ce qui facilite la collaboration et le partage de projets.
    4. Optimisation des ressources:
      • Allocation précise: Les ressources (CPU, mémoire, disque) peuvent être allouées de manière précise à chaque machine virtuelle, optimisant ainsi l’utilisation du matériel.
    5. Sécurité:
      • Isolation: Chaque machine virtuelle est isolée des autres, ce qui limite les risques de propagation de malwares ou de conflits.
    En résumé, les machines virtuelles offrent un environnement flexible, sécurisé et reproductible pour apprendre et expérimenter les concepts de l'administration système Linux.

  • C’est quoi SSH (et OpenSSH)

    Le protocole SSH : Une passerelle sécurisée vers les systèmes distants

    Qu’est-ce que le protocole SSH ?

    Le protocole SSH (Secure Shell) est un standard réseau qui permet d'établir des connexions sécurisées à distance vers d'autres ordinateurs. Il offre un moyen crypté de se connecter à un serveur distant, d'exécuter des commandes, de transférer des fichiers et de gérer des systèmes à distance.

    Les principales caractéristiques du protocole SSH :

    • Sécurité: Le trafic est entièrement chiffré, ce qui protège les données contre les interceptions.
    • Authentification: Plusieurs méthodes d’authentification sont disponibles, notamment les mots de passe et les clés SSH.
    • Flexibilité: Le protocole SSH offre de nombreuses fonctionnalités, telles que le transfert de fichiers (SCP et SFTP), le tunneling, et la configuration dynamique de ports.

    OpenSSH : L’implémentation la plus populaire du protocole SSH

    Qu’est-ce qu’OpenSSH ?

    OpenSSH est une implémentation libre et open-source du protocole SSH. C'est le logiciel SSH le plus couramment utilisé sur les systèmes Unix-like (Linux, BSD, etc.). Il est également disponible pour Windows.

    Les avantages d’OpenSSH :

    • Libre et gratuit: OpenSSH est distribué sous licence libre, ce qui signifie qu’il est gratuit à utiliser et à modifier.
    • Sécurité: OpenSSH est régulièrement mis à jour pour corriger les vulnérabilités, ce qui en fait une solution sécurisée.
    • Fonctionnalités étendues: OpenSSH offre de nombreuses fonctionnalités supplémentaires par rapport au protocole SSH de base.
    • Largement utilisé: Étant l’implémentation la plus populaire, OpenSSH est bien supporté et documenté.

    En résumé:

    • SSH est un protocole qui définit les règles pour établir des connexions sécurisées.
    • OpenSSH est une implémentation concrète de ce protocole, offrant un ensemble d’outils pour se connecter et gérer des systèmes distants de manière sécurisée.
    En conclusion, OpenSSH est un outil essentiel pour tout administrateur système ou développeur qui travaille avec des systèmes distants. Il offre une connexion sécurisée, flexible et puissante, facilitant ainsi la gestion et l'administration de vos infrastructures informatiques.

  • C’est quoi Crontab

    Crontab permet de planifier l’exécution automatique de tâches à des horaires ou fréquences spécifiques. Il utilise une syntaxe de fichier de configuration appelé cron table pour spécifier ces tâches.

    Fonctionnement de crontab :

    1. Commandes planifiées : Vous pouvez planifier des scripts ou des commandes à exécuter à intervalles réguliers comme toutes les heures, tous les jours, chaque semaine, etc.
    2. Format du fichier crontab : Chaque ligne dans le fichier crontab spécifie une tâche à exécuter, en utilisant une syntaxe qui indique le moment où la tâche doit être lancée.

    Syntaxe d’une ligne crontab :

    Une ligne dans le fichier crontab suit le format suivant :

    * * * * * commande
- - - - -
| | | | |
| | | | +-- Jour de la semaine (0 - 7) (dimanche = 0 ou 7)
| | | +---- Mois (1 - 12)
| | +------ Jour du mois (1 - 31)
| +-------- Heure (0 - 23)
+---------- Minute (0 - 59)

    Exemple d’utilisation :

    Voici un exemple de tâche planifiée dans un fichier crontab :

    30 8 * * 1-5 /usr/local/bin/monitoring.sh

    Cette ligne indique que le script /usr/local/bin/monitoring.sh doit être exécuté à 8h30 du matin, du lundi au vendredi (jours de la semaine 1 à 5).

    Explication des champs :

    1. Minute : De 0 à 59. (Quand la commande doit être lancée dans la minute).
    2. Heure : De 0 à 23. (L’heure de la journée).
    3. Jour du mois : De 1 à 31. (Quel jour du mois).
    4. Mois : De 1 à 12. (Quel mois de l’année).
    5. Jour de la semaine : De 0 à 7, où 0 et 7 sont dimanche. (Quel jour de la semaine).

    Symboles spéciaux :

    • * (astérisque) : signifie « chaque » unité de temps. Par exemple, * * * * * exécute la commande toutes les minutes, toutes les heures, tous les jours, etc.
    • , (virgule) : permet de spécifier plusieurs valeurs. Par exemple, 5,10 dans le champ des minutes signifie que la commande s’exécutera à la 5e et 10e minute de chaque heure.
    • - (tiret) : permet de spécifier une plage de valeurs. Par exemple, 1-5 dans le champ des jours de la semaine indique que la tâche s’exécutera du lundi au vendredi.
    • / (barre oblique) : permet de spécifier un intervalle. Par exemple, */15 dans le champ des minutes signifie que la tâche s’exécutera toutes les 15 minutes.

    Exemples

    1. Exécuter un script tous les jours à 2h30 du matin :
      • 30 2 * * * /usr/local/bin/monitoring.sh
    2. Exécuter une commande chaque lundi à midi :
      • 0 12 * * 1 /path/to/command
    3. Exécuter un script toutes les 10 minutes :
      • */10 * * * * /usr/local/bin/monitoring.sh

    Gestion du crontab :

    • Visualiser le crontab : Utilisez la commande crontab -l pour lister les tâches planifiées de l’utilisateur courant.
    • Modifier le crontab : Utilisez crontab -e pour éditer le fichier crontab.
    • Supprimer le crontab : Utilisez crontab -r pour supprimer toutes les tâches planifiées de l’utilisateur courant.

    Résumer

    Crontab est un outil pour l’automatisation des tâches sur des systèmes Unix/Linux. Il est utile pour les administrateurs système et les développeurs afin de gérer les sauvegardes, les mises à jour, les nettoyages ou autres tâches récurrentes de manière programmée.

  • Le fameux monitoring.sh

    Voici un exemple de script correspondant à la consigne ‘monitoring.sh’ :

    /usr/local/bin/monitoring.sh
    #!/bin/bash
arc=$(uname -a)
pcpu=$(grep "physical id" /proc/cpuinfo | sort | uniq | wc -l) 
vcpu=$(grep "^processor" /proc/cpuinfo | wc -l)
fram=$(free -m | awk '$1 == "Mem:" {print $2}')
uram=$(free -m | awk '$1 == "Mem:" {print $3}')
pram=$(free | awk '$1 == "Mem:" {printf("%.2f"), $3/$2*100}')
fdisk=$(df -BG | grep '^/dev/' | grep -v '/boot$' | awk '{ft += $2} END {print ft}')
udisk=$(df -BM | grep '^/dev/' | grep -v '/boot$' | awk '{ut += $3} END {print ut}')
pdisk=$(df -BM | grep '^/dev/' | grep -v '/boot$' | awk '{ut += $3} {ft+= $2} END {printf("%d"), ut/ft*100}')
cpul=$(top -bn1 | grep '^%Cpu' | cut -c 9- | xargs | awk '{printf("%.1f%%"), $1 + $3}')
lb=$(who -b | awk '$1 == "system" {print $3 " " $4}')
lvmu=$(if [ $(lsblk | grep "lvm" | wc -l) -eq 0 ]; then echo no; else echo yes; fi)
ctcp=$(ss -Ht state established | wc -l)
ulog=$(users | wc -w)
ip=$(hostname -I)
mac=$(ip link show | grep "ether" | awk '{print $2}')
cmds=$(journalctl _COMM=sudo | grep COMMAND | wc -l)
wall "	#Architecture: $arc
	#CPU physical: $pcpu
	#vCPU: $vcpu
	#Memory Usage: $uram/${fram}MB ($pram%)
	#Disk Usage: $udisk/${fdisk}Gb ($pdisk%)
	#CPU load: $cpul
	#Last boot: $lb
	#LVM use: $lvmu
	#Connections TCP: $ctcp ESTABLISHED
	#User log: $ulog
	#Network: IP $ip ($mac)
	#Sudo: $cmds cmd"

    Commandes de base et leur rôle dans le script:

    Free : 

    Affiche des informations détaillées sur la mémoire système (mémoire totale, utilisée, libre). Dans le script, elle est utilisée pour calculer l'utilisation de la mémoire.

    Uname : 

    Fournit des informations générales sur le système, comme le nom du système d'exploitation, le nom de la machine et la version du noyau. Elle sert à obtenir l'architecture du système dans notre script.

    Grep : 

    Permet de rechercher des lignes contenant une chaîne de caractères spécifique dans un fichier. Il est utilisé pour extraire des informations pertinentes de fichiers comme /proc/cpuinfo et /proc/meminfo

    Awk : 

    Un outil de traitement de texte puissant. Il permet d'extraire des champs spécifiques de lignes, de faire des calculs et de formater les données. Dans le script, il est utilisé pour extraire des valeurs numériques à partir des sorties de free et df.

    Df : 

    Affiche l'espace disque utilisé et disponible sur les systèmes de fichiers. Il est utilisé pour calculer l'utilisation du disque.

    Wc : 

    Compte les lignes, les mots et les caractères dans un fichier. Il est utilisé pour compter le nombre de lignes ou de mots dans la sortie d'autres commandes.

    Sort : 

    Trie les lignes d'un fichier. Il est utilisé pour trier les résultats avant de les traiter avec 'uniq'

    Uniq : 

    Supprime les lignes en double consécutives dans un fichier trié.

    Xargs : 

    Construit et exécute des commandes à partir de la sortie d'une autre commande.

    Who : 

    Affiche les utilisateurs actuellement connectés.

    Ss : 

    Affiche des statistiques sur les sockets réseau.

    Hostname – I : 

    Affiche le nom d'hôte du système.

    Journalctl : 

    Permet de consulter le journal système.

    Cut : 

    Extrait des colonnes spécifiques d'un fichier.

    Top : 

    Affiche les processus en cours d'exécution et leurs ressources utilisées

    Lsblk : 

    Affiche une liste des blocs de stockage (disques, partitions, volumes logiques).

    Wall :

    Envoie un message à tous les utilisateurs connectés.

    Le script utilise ces commandes pour :

    1. Collecter des informations système: Les commandes comme uname, free, df, top, lsblk, who, ss, hostname, et journalctl sont utilisées pour recueillir des données sur le matériel, les logiciels et l’état du système.
    2. Traiter les données: Les commandes grep, awk, sort, uniq, et cut sont utilisées pour extraire, filtrer et formater les données collectées.
    3. Afficher les résultats: La commande wall est utilisée pour afficher les informations collectées et traitées à tous les utilisateurs connectés.
